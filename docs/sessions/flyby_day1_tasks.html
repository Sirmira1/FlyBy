BACKEND — AUTH ROUTES

POST /api/auth/register — takes email, password, username. Creates the Supabase auth user and inserts a row into your users table. Validates that the username is unique.
1.Fill out routes/auth.js register endpoint
2.Check username uniqueness before creating account
3.Test with Postman or curl — create a test account

Build the login route backend ~15 min
POST /api/auth/login — takes email and password, returns a session token. This token is what the mobile app will store and send with every request.
1.Fill out routes/auth.js login endpoint
2.Return the session access_token and user object
3.Test login with the account you created above
